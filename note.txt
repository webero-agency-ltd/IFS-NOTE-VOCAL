
1 - ) Restructureation du dossier de l'application 

- séparation du dossier native a l'applicaion ( Comportement GLOBALE ) 
- séparation du dossier relative a un fonctionnalité de l'app ( ex : Scrapin DATA, ... ) 

cella va facilité la recherche de bug de l'application, facilité aussi l'ajout de nouvelle foctionnalité 

/backend : 	pour le script en background de l'application 

/content : 	pour le script de contenu de l'application ( injection de script ex :ouvre une page et inject ce script )
			si on trouve une solution pour rendre différent script en fonction de l'action ex : (scrapin.js ou groupe.js ) cella va encore plus rendre l'app plus maintenable. mais le faire coté webpack 

/options : 	pour le script de vue qui va rendre la vue de l'application ( ex : tableaux des listé scrapé )
			de ce fait cette application doivent etre en concordance avec les code des module 

/popup :	Page des popups javascirpt dans google chrome extenssion seulement. 

/lib : 		La librari utilisé par l'application en générale et qui pourait etre utilisé dans différent zone ex : 
			la base de donner, le lang , les fonction utiles, ... 

/module : 	C'est dans cette partie que l'on doivent metre tout les codes de l'application. MAIS MAIS MAIS :
			tout les code qui on écrit ici font référence a ( /backend, /content, /Options, /popup ) et tout les donners de cette app seront structuré de la meme sorte 

			- trouver une solution ou si on est dans le module, on crée une fonction ou un lib et qui pourait etre utilisé dans tout les autres module ( le ca on on crée deux module différent et que l'utilisation du deuxième module nessaicite l'utilisation d'un fonction dans le deusième module )


injection.js : fichier javascript pour l'injection de tout les modules dans les fichier qui ex : dans module on a un module "node/check" et ceci a un code en backend donc on fait ca ICI
